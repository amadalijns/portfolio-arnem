SOC

In mijn homelab heb ik een SOC (Security Operations Center) opgezet om beter te begrijpen hoe je systemen actief kunt monitoren en beveiligen. Hiervoor heb ik verschillende tools met elkaar laten samenwerken, waaronder een SIEM, threat intel integraties en active response. De focus ligt op het verzamelen en analyseren van logs, het detecteren van verdachte activiteiten en het automatisch reageren op bepaalde dreigingen.

Deze setup helpt me om meer praktijkervaring op te doen met alerting, incident response en threat hunting.

Cloud

AWS | Automatisering | Pipelines

Tijdens mijn opleiding heb ik op het gebied van cloud engineering voornamelijk gewerkt met Amazon Web Services (AWS), waarbij we via verschillende opdrachten praktijkervaring opdeden met cloudplatformen en Infrastructure as Code (IaC).

Een van de eerste opdrachten was het maken van een Python-script dat automatisch een EC2-instance (virtuele machine in AWS) kon aanmaken. De gebruiker kon zelf kiezen welk besturingssysteem hij wilde en het script zorgde er vervolgens voor dat de instance correct werd opgezet in AWS.

Naarmate de opleiding vorderde, werden de projecten complexer. Zo kregen we de opdracht om een multi-cloudapplicatie te bouwen met Terraform, waarbij zowel AWS als Azure werd gebruikt. We moesten het spel Galgje hosten in de cloud, waarbij de API draaide in een containeromgeving op Azure en de frontend werd gehost op AWS binnen een Auto Scaling Group (ASG).

In project Frank de Pratende Bank lag mijn verantwoordelijkheid bij het volledige deploymentproces via GitLab Pipelines. Elke keer als er nieuwe code werd gepusht, werd deze gecontroleerd op fouten en beveiligingsproblemen. Daarna werd de applicatie omgevormd tot een Docker-image, getest, en als alles in orde was automatisch uitgerold naar de productieomgeving in Azure.
